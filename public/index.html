Core

Adds a new card

POST /api/v1/card/addCard

Adds one or more cards to the database if they don't already exist. If the cards exist, it returns the corresponding card hashes. Only 15 credit cards can be added based on TSYS limitation, if 15+ credit cards are send, then the API will throw an error. The last 4 digits of each of the credit card numbers should be unique. If they are not unique, then the API will throw an error. In both the above mentioned cases, if the debit cards are present in the list, then those debit cards will be added into the system and the response will contain hashes for the debit cards and error messages for the credit cards.

object
number: integer (up to 19 chars)

Card number to be added

name: string

Card Name

statementDate: integer

Due date of the card statement, recommended when using limits for credit.

userId: string

Used by issuer to relate cards to a customer. If the sync flag is true then userId is a required field. Note that userId is stored for debit cards automatically and is only stored for credit cards if the full TSYS registration is completed. Furthermore, it is highly suggested to use the same userId for a previously TSYS-registered credit card. Failure to do so by using a different userId during re-registration may cause unexpected behavior on TSYS system with respect to using the latest userId.

sync: boolean

Register card with TSYS. If set to true, then the card will be registered with TSYS


X-Auth-Token: string
in header

The authorization token sent for every API request

Enable or disable payment channels

POST /api/v1/cardSettings/changePaymentChannels

Enable or disable the different payment channel options of a card. Any changes made to the channel settings for credit cards are sent over to the TSYS OnDot system. By default, all payment channels are enabled for card use. If a channel is disabled, any transaction from that card for the channel is blocked. The standard ones are: ATM, POS, ECOM, and MOTO.

POST /api/v1/cardSettings/getPaymentChannels

Respond with the card's payment channels. The TSYS OnDot system is called in order to retrieve the latest payment channel settings of a credit card.
